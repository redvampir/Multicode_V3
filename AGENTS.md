# Инструкции по стилю репозитория

## Общие требования

- Перед коммитом обязательно запускайте `pre-commit run --all-files` и исправляйте все выявленные замечания.
- Новые инструменты для проверки стиля документируйте в README.md в разделах «Установка зависимостей» и «Проверка кода».
- При обновлении конфигураций следите за совместимостью с существующими скриптами в `scripts/`.

## Markdown (`*.md`)

- Форматирование выполняется через Prettier (`npx prettier --write`).
- Заголовки строятся иерархично без пропусков уровней, списки — с последовательной вложенностью.
- Ссылки на другие документы указываются относительными путями в каталоге `Документы/`.

## Скрипты

- Bash-скрипты хранятся в `scripts/` и проходят проверку `bash -n`.
- При изменении merge-драйверов не забывайте обновлять документацию по конфликтам в `Документы/`.
